## Adjustment made

Fixed a data-quality blind spot in current edge scans.

Root cause: synthetic sessions use their session date as the index. Before the fix, `_edge_data_quality()` measured staleness from that synthetic session date and did not penalize truly stale complete data. This could either overstate staleness from anchor-date sessions or keep genuinely stale data at `quality_score=1.0`.

Change:

- `scanner/data/synthetic_sessions.py` now stores `latest_source_timestamp` from the source intraday bars in `synthetic.attrs`.
- `scanner/main.py` now prefers `latest_source_timestamp` when computing staleness and reduces `quality_score` after a 30-minute grace window plus any configured data delay.
- `scanner/tests/test_edge_cli_units.py` now verifies stale complete bars lose quality, fresh complete bars stay pristine, and synthetic sessions use the latest source timestamp.
